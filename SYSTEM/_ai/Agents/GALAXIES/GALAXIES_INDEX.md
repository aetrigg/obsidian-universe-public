# Galaxies Index

This document provides an overview of all active galaxies (projects/areas of interest) in Andrea's universe, as well as the root nebulae (system folders) that support the entire vault.

---

## üåå What Makes a Galaxy?

**Galaxies** are major projects or areas of interest that Andrea is actively working on. They are distinct from root nebulae (system folders) in that they represent creative work, personal development, or ongoing projects rather than organizational infrastructure.

### Galaxy Formation Requirements
- Must be hyperfixated on the topic for a **minimum of 3 months** before creating a new galaxy
- This prevents galaxy sprawl and ensures galaxies represent sustained interests
- Galaxies can go dormant but aren't deleted - they may become active again

### Galaxy vs. Root Nebula
- **Galaxies** = Projects, creative work, areas of development (e.g., 5th of June, Digital Hobonichi)
- **Root Nebulae** = System folders that support the entire universe (e.g., INBOX, SYSTEM, OBSERVATORIES)

Most galaxies have their own dedicated root nebula (folder) at the vault's root level. The exception is the Obsidian galaxy, which is meta-organizational and doesn't need its own folder.

## üöÄ Creating New Galaxies

If Andrea develops a new sustained interest (3+ months), creating a new galaxy involves:

1. Create dedicated root nebula (folder) for the galaxy
2. Add galaxy to this index file
3. Create detailed galaxy documentation in `GALAXIES/[galaxy-name].md`
4. Consider if new INBOX sub-folder needed, or use existing categorization
5. Update tag routing in [[MISSION_PROTOCOLS]] if new constellation needed
